Line Chart Component

A flexible, feature-rich line chart component for Svelte 5

Svelte 5 TypeScript Negative Values Auto Ticks

Key Features

Multiple Lines

Display up to 5 lines simultaneously with automatic color assignment and interactive legend.

Negative Values Support

Automatic handling of negative data with zero reference line and enhanced scaling.

Auto-Doubled Ticks

Automatically increases tick density when negative values are detected for better granularity.

Responsive Design

Automatically adjusts to container size with smooth animations and transitions.

Value Labels

Optional display of values above each data point for precise reading.

Accessibility

Full keyboard navigation, comprehensive ARIA support, screen reader compatibility, and alternative data table access.

Props Reference

Prop Type Default Description
lines LineData[] [] Array of line data objects (max 5 lines)
xKey string 'date' Key to use for x-axis values
yKey string 'value' Key to use for y-axis values
title string 'Multi Line Chart' Chart title
showLegend boolean true Whether to show the legend
height number 400 Chart height in pixels
showValues New boolean false Show value labels above each data point
hasTooltip New boolean true Enable/disable tooltips on hover
yTickCount New number 5 Number of ticks on Y-axis
doubleTicksForNegatives New boolean true Automatically double tick count when negative values detected
dateFormat string 'MMM dd' Date format for display (using date-fns format)
inputDateFormat string | null null Expected input date format for parsing

Basic Usage

import { MultiLineChart } from '$lib'; const lines = [ { id: 'desktop', label: 'Desktop Sales', color: '#3b82f6', data: [ { month: 'Jan', sales: 1000 }, { month: 'Feb', sales: 3200 } // ... more data ] } ]; // Basic usage <MultiLineChart {lines} xKey="month" yKey="sales" title="Product Performance" />

Negative Values Support

Automatic Features

When negative values are detected, the component automatically adds a zero reference line and can double the tick count for better granularity around the zero line.

const profitLossData = [ { id: 'product_a', label: 'Product A', color: '#3b82f6', data: [ { month: 'Jan', profit: -1200 }, { month: 'Feb', profit: -800 }, { month: 'Mar', profit: 200 }, { month: 'Apr', profit: 1500 } ] } ]; <MultiLineChart lines={profitLossData} xKey="month" yKey="profit" title="Profit/Loss Analysis" yTickCount={5} doubleTicksForNegatives={true} />

Advanced Configuration

With Value Labels

<MultiLineChart {lines} showValues={true} showLegend={true} />

Custom Tick Count

<MultiLineChart {lines} yTickCount={8} doubleTicksForNegatives={false} />

Disable Tooltips

<MultiLineChart {lines} hasTooltip={false} />

Custom Date Format

<MultiLineChart {lines} dateFormat="MMM dd, yyyy" inputDateFormat="dd/MM/yyyy" />

Best Practices

Data Guidelines

• Limit to 5 lines maximum for readability
• Ensure data consistency across lines
• Use meaningful x-axis intervals

Visual Design

• Use high contrast colors
• Provide descriptive legend labels
• Consider colorblind accessibility

Negative Values

• Keep auto-doubled ticks enabled
• Use contextual colors (red/green)
• Provide clear zero reference

Accessibility

• Full keyboard navigation support
• Screen reader friendly with ARIA
• Alternative data table access
• Live region announcements

Performance

• Use value labels sparingly
• Disable tooltips for dense data
• Optimize tick count for your data

Accessibility Features

This chart component is designed with accessibility as a core feature, providing multiple ways to interact with and understand the data:

Keyboard Navigation

Tab: Navigate between legend items
Enter/Space: Toggle line highlighting
Escape: Clear all highlights

Screen Reader Support

• Comprehensive ARIA labels and descriptions
• Live region announcements for interactions
• Chart summary and data point information

Alternative Data Access

• Optional accessible data table view
• Proper table headers and structure
• Toggle between chart and table views

Visual Accessibility

• High contrast colors and tooltips
• Clear focus indicators
• Responsive design for all screen sizes

WCAG Compliance

This component meets WCAG 2.1 AA standards for accessibility, providing perceivable, operable, understandable, and robust chart interactions for all users.
